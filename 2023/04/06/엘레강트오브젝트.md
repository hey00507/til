# 엘레강트 오브젝트
- 누군가 내 코드를 이해할 수 없다면, 그 원인은 나에게 있다.
- 코드의 유지보수성을 향상시키자.

## 유지보수성이 높은 코드?
- 길이가 짧다.
- 소화하기 쉽다.
- 모듈성이 높다.
- 응집도가 높다.



### 클래스는 객체의 팩토리이다.
- 클래스를 그저 코드의 덩어리로 보면 안된다.
- 필요할 때 객체를 꺼내거나, 필요하지 않은 객체를 반환하는 객체의 창고 역할을 한다.
- 즉, 클래스는 객체의 능동적 관리자이며, 객체의 어머니이다.


### 클래스의 명명법
- 클래스안의 객체들이 무엇을 하는지 살피고, 기능에 기반해 이름을 짓는 법 -> 잘못됨
- 클래스가 무엇을 하는지? 가 아닌, 무엇인지! 에 초점을 맞추자.
- 객체는 연결장치가 아니다.
